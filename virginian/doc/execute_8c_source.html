00005  * @brief Execute the virtual machine using its stored statement
00006  *
00007  * Execute the opcodes that have been stored in the passed virtual machine
00008  * struct, choosing the execution location based on the options set in the
00009  * virginian struct. This function executes the top-level opcodes that must be
00010  * handled serially. Like the CPU virtual machine, this function uses a jump
00011  * table to access code blocks with opcodes, rather than a switch statement. The
00012  * jump table is simply an array of label addresses that is accessed using the
00013  * integer value of each opcode, then jumped to with a goto. This function is
00014  * also responsible for fetching the first data tablet of each loaded table and
00015  * allocating the first result tablet, then releasing the final locks on the
00016  * data and result tablet pointers at the end of the query, probably after the
00017  * pointers have been altered by the lower-level execution functions.
00018  * virg_vm_gpu() or virg_vm_cpu() is chosen based on the options in the
00019  * virginian struct, and there is currently no capability to handle both
00020  * simultaneously, as this would probably involve a more complex threading
00021  * system.
00022  *
00023  * @param v     Pointer to the state struct of the database system
00024  * @param vm    Pointer to the context struct of the virtual machine
