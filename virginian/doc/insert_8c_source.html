00004  * @ingroup table
00005  * @brief Insert a row into a table
00006  *
00007  * Insert a new row by adding it to the end of a table. This function locates
00008  * the tablet where we have set the write_cursor. If the tablet is full, we
00009  * attempt to add more row space with virg_tablet_addrows(), and if we can't,
00010  * then we move onto the next tablet in the tablet string. The key and data
00011  * arguments are passed as pointers to their buffer because the size of their
00012  * variable types is unknown. The data buffer should contain all the columns in
00013  * order immediately adjacent to each other. For example, the following code
00014  * adds a row to a table with an integer key, and an integer, double, and float
00015  * column:
00016  *
00017  * @code
00018  * int i = 150;
00019  * char buff[sizeof(int) + sizeof(double) + sizeof(float)];
00020  * char *x = &buff[0];
00021  * ((int*)x)[0] = 100;
00022  * x += sizeof(int);
00023  * ((double*)x)[0] = 100.0;
00024  * x += sizeof(double);
00025  * ((float*)x)[0] = 100.0;
00026  * virg_table_insert(v, table_id, &i, &buff[0], NULL);
00027  * @endcode
00028  *
00029  * @param v             Pointer to the state struct of the database system
00030  * @param table_id      Table in which to insert the row
00031  * @param key           Buffer containing the key value for this row
00032  * @param data          Buffer containing the row data
00033  * @param blob          Buffer containing variable size data associated with the
00034  * key (unimplemented)
00035  * @return VIRG_SUCCESS or VIRG_FAIL depending on errors during the function
00036  * call
00037  */
