Detailed Description

Lower-level data parallel CPU virtual machine context.

This is the cache-efficient data structure used to store intermediate information for CPU virtual machine execution. The VIRG_CPU_SIMD macro specifies the size of the row blocks that are processed in one step on the CPU. Note that the registers are blocked for data-parallel execution, so the data used by different rows in the same register location is adjacent. This is accomplished with the array of unions. Also note that the type and stride of each register only needs to be stored once per register, because everything is SIMD in a row block.
