00006  *
00007  * This function writes the tablet in a tablet slot to an area of the on-disk
00008  * database file. It is not thread-safe, so locking of the tablet slot array and
00009  * the tablet should be performed in a multi-threaded environment. This function
00010  * handles updating the disk slot meta information about where tablets are
00011  * stored.
00012  *
00013  * This function is somewhat complex because of the variable-sized disk meta
00014  * information. The on-disk database stores a fixed-size block of meta
00015  * information that corresponds to a virg_db struct. The listing of the tablet
00016  * IDs in the database file is of a variable size because we can have an
00017  * arbitrarily large number of tablets stored in the file. After the tablet
00018  * listing, we store actual tablets. Note that this variable-size tablet list,
00019  * or tablet info block, is stored in memory while the database is open and
00020  * written to disk when the database closes. Under this scheme, we often exceed
00021  * the currently allocated size of this tablet list, in which case we must
00022  * allocate a larger area and copy the contents to the new allocation. Problems
00023  * arise, however, because this new larger size may intersect with the first
00024  * tablet in the database file, in which case we must move this tablet to the
00025  * back of the list, and move on disk if it is not loaded into memory, to make
00026  * more room for our tablet list in the database file. A significant portion of
00027  * this function is devoted to this edge case.
00028  *
00029  * @param v Pointer to the state struct of the database system
